ArchivesSpace 4.0 represents a major leap forward for the platform. We've focused on three key areas: performance, extensibility, and the day-to-day experience for staff.

## Performance improvements

The backend has been optimized for larger datasets. Index rebuilds run faster, and search queries show noticeable improvement on repositories with hundreds of thousands of records. We've also reduced memory usage during batch operations.

## Plugin support

The plugin architecture has been expanded. Developers can now hook into more lifecycle events and extend additional parts of the application. Documentation for plugin development is available in the [Development](/development/dev) section of these docs.

## Staff interface updates

The staff interface includes a refreshed layout and improved keyboard navigation. Form validation is more responsive, and error messages are clearer. We've also made progress on accessibility compliance for screen readers and other assistive technologies.

## What's next

We're already working on 4.1. If you have feedback on 4.0, please reach out on the [ArchivesSpace Community Forum](https://archivesspace.atlassian.net/wiki/spaces/ADC/overview) or open an issue on GitHub.
